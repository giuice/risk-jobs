## Interaction Contract

### VIS-02: Neon Glow Text-Shadow

- Applied as static `text-shadow` declarations on `.page-title` and `.countdown-value`
- Applied as static `text-shadow` on `.results-card-heading` (cyan glow)
- Never animated inside `@keyframes`
- Custom property values stored in `--neon-glow-green`, `--neon-glow-pink`, `--neon-glow-cyan`

### VIS-03: Scanline Overlay

- Implemented as `body::after` pseudo-element
- `position: fixed; inset: 0` — covers full viewport during scroll
- `pointer-events: none` — clicks and hover states pass through
- `z-index: 9999` — above all content
- `-webkit-transform: translateZ(0); transform: translateZ(0)` — forces compositing layer, prevents iOS Safari flicker
- Dark band opacity: exactly `0.18` — must not exceed `0.30`
- Band height: 4px total (2px transparent, 2px dark)

### VIS-04: Glitch / CRT Reveal Animation

- Applied exclusively to `#results-view .page-title` — not to card headings (too small, reads as noise)
- Triggered once when `showResults()` runs — not on page load, not on input view
- Mechanism: `::before`/`::after` pseudo-elements with `content: attr(data-text)`, colored `--neon-cyan` and `--neon-pink` respectively
- Animation: `clip-path: inset()` + `transform: translateX()` via `@keyframes glitch-top` and `@keyframes glitch-bot`
- Duration: exactly 0.6s — `steps(4)` timing, `animation-iteration-count: 1`, `fill-mode: forwards`
- Only `clip-path` and `transform` animated — GPU-compositor-safe, no layout reflow
- JS trigger in `showResults()`: remove `glitch-active`, `requestAnimationFrame`, add `glitch-active`
- JS cleanup in `resetResultState()`: remove `glitch-active`
- `data-text` attribute must be set in `showResults()` before adding class

### VIS-05: Mobile Responsive at 375px

Breakpoint: `@media (max-width: 480px)`

Applied changes:
- `body { padding: 12px }` — reduces from 20px
- `.container { padding: 12px }` — reduces from 20px
- `.card-grid { grid-template-columns: repeat(auto-fill, minmax(130px, 1fr)); gap: 8px }` — reduces from 160px min-width
- `.card { padding: 12px 8px }` — reduces from 16px 12px
- `.card .card-name { font-size: 0.78rem }` — reduces from 0.85rem
- `.cta-btn { padding: 14px 24px; font-size: 1rem; width: 100% }` — full width at mobile
- `.exp-btn { min-width: 120px; padding: 12px 14px; font-size: 0.8rem }` — reduces from 140px min-width
- `.page-title { font-size: 1.3rem; letter-spacing: 1px }` — reduces from 1.8rem
- `.countdown-value { font-size: 2rem }` — reduces from 3rem
- `.countdown-grid { grid-template-columns: repeat(3, minmax(0, 1fr)); gap: 8px }` — overrides existing 640px single-column rule with 3-column at 480px

Existing breakpoint at `@media (max-width: 640px)` collapses countdown grid to 1 column — Phase 3 overrides this only at 480px and below.

### Reduced Motion (Accessibility)

`@media (prefers-reduced-motion: reduce)`:
- `animation: none` on `.page-title.glitch-active::before` and `::after`
- `transition-duration: 0.001ms` on `.results-panel.results-visible .results-reveal` — near-instant, preserves state change without motion
- Static effects (neon glow, scanlines) remain active — they are not motion-based

## Anti-Patterns (Executor Reference)

- Do NOT animate `text-shadow` or `box-shadow` inside `@keyframes` — forces paint every frame, explicitly prohibited by VIS-02 success criteria
- Do NOT use `position: absolute` on the scanline overlay — scroll will expose the page below the overlay; must be `position: fixed`
- Do NOT run glitch animation infinitely — one cycle per results reveal only (`animation-iteration-count: 1`)
- Do NOT omit `pointer-events: none` on `body::after` — overlay will block all card and button interaction
- Do NOT omit `data-text` attribute before adding `glitch-active` — pseudo-elements produce blank colored blocks
- Do NOT set `overflow: hidden` on `body` — breaks `position: fixed` in iOS Safari; use `overflow-x: hidden` on a wrapper if needed
- Do NOT remove static effects (neon glow, scanlines) under `prefers-reduced-motion` — only keyframe animations are suppressed
