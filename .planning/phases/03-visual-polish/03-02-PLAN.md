Add the mobile layout CSS section inside the `<style>` block, after the `/* ---- Reduced Motion ---- */` section and before the closing `</style>` tag.

```css
/* ---- Mobile Layout (VIS-05) ---- */
@media (max-width: 480px) {
  body {
    padding: 12px;
  }

  .container {
    padding: 12px;
  }

  .card-grid {
    grid-template-columns: repeat(auto-fill, minmax(130px, 1fr));
    gap: 8px;
  }

  .card {
    padding: 12px 8px;
  }

  .card .card-name {
    font-size: 0.78rem;
  }

  .cta-btn {
    padding: 12px 24px;
    font-size: 1rem;
    width: 100%;
  }

  .exp-btn {
    min-width: 120px;
    padding: 12px 14px;
    font-size: 0.8rem;
  }

  .page-title {
    font-size: 1.3rem;
    letter-spacing: 1px;
  }

  .countdown-value {
    font-size: 2rem;
  }

  .countdown-grid {
    grid-template-columns: repeat(3, minmax(0, 1fr));
    gap: 8px;
  }
}
```

**Why these specific values:**

- `body { padding: 12px }` + `.container { padding: 12px }` = 48px total horizontal gutters at 375px, leaving 327px usable width. Down from 80px (body 40px + container 40px) which left only 295px.
- `.card-grid { minmax(130px, 1fr) }` — two columns at 327px usable width: 2 x 130px + 8px gap = 268px, fits comfortably. The old `minmax(160px, 1fr)` needed 332px minimum and caused horizontal scroll.
- `.card { padding: 12px 8px }` — tighter padding saves ~16px per card, keeping card content from wrapping excessively.
- `.card .card-name { font-size: 0.78rem }` — Label-variant at mobile. Prevents long profession names from wrapping to 3+ lines.
- `.cta-btn { width: 100% }` — full-width button is a larger tap target on mobile and eliminates the risk of horizontal overflow from 48px side padding.
- `.exp-btn { min-width: 120px }` — reduced from 140px. At 327px usable width, `flex-wrap: wrap` already handles layout, but 120px allows 2 buttons per row with breathing room.
- `.page-title { font-size: 1.3rem }` — Heading role at mobile. 1.8rem is too large at 375px with neon glow.
- `.countdown-value { font-size: 2rem }` — Display role at mobile. 3rem causes the countdown grid to overflow.
- `.countdown-grid { repeat(3, minmax(0, 1fr)); gap: 8px }` — OVERRIDES the existing 640px breakpoint that collapses to 1 column. At 480px and below, three columns at 2rem font are compact enough. `minmax(0, 1fr)` prevents minimum content size from forcing overflow. This rule appears AFTER the 640px rule in the cascade, and since 480px < 640px, both match — but this rule's properties override by cascade order.

**Important:** The 480px media query must appear AFTER the existing `@media (max-width: 640px)` rule in the source order. Since the 480px breakpoint is more specific (narrower range), placing it after ensures its `.countdown-grid` override takes precedence when both media queries match.
