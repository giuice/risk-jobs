  <action>
Deploy the repo to GitHub Pages:

1. **Ensure all changes are committed:**
   ```bash
   git add index.html
   git status
   ```

2. **Push to the `main` branch** (GitHub Pages default branch):
   - The repo's default branch is `main` but current work is on `master`
   - Check if `main` branch exists: `git branch -a | grep main`
   - If `main` doesn't exist locally: `git checkout -b main` then push
   - If `main` exists: `git checkout main`, merge master, then push
   - `git push origin main`

3. **Enable GitHub Pages via the GitHub CLI:**
   ```bash
   gh api -X PUT repos/giuice/risk-jobs/pages \
     -f source='{"branch":"main","path":"/"}' \
     --input - <<< '{"source":{"branch":"main","path":"/"}}'
   ```
   If Pages is already enabled, this may return an error — that's OK. Alternatively:
   ```bash
   gh api -X POST repos/giuice/risk-jobs/pages \
     -f "source[branch]=main" -f "source[path]=/"
   ```
   If already configured, update instead:
   ```bash
   gh api -X PUT repos/giuice/risk-jobs/pages \
     -f "source[branch]=main" -f "source[path]=/"
   ```

4. **Wait for deployment** (Pages can take 1-3 minutes):
   ```bash
   sleep 30
   curl -s -o /dev/null -w "%{http_code}" https://giuice.github.io/risk-jobs/
   ```
   If not 200, wait another 30s and retry. GitHub Pages deployments can take up to 5 minutes on first setup.

5. **Verify the page responds:**
   ```bash
   curl -s https://giuice.github.io/risk-jobs/ | grep -c 'AI Job Death Clock'
   ```
   Should return at least 1.
